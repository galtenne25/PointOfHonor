# מאחורי הקלעים: איך נבנתה "נקודת ציון" (Vibe Coding & Production)

היי! הוספתי את המסמך הזה כדי לשתף קצת על תהליך העבודה מאחורי הקלעים, ואיך השתדלתי לקחת את הפרויקט הזה צעד אחד קדימה מעבר לדרישות הרגילות של עבודה אקדמית.

## 1. לבנות מוצר אמיתי, לא רק אבטיפוס
היה לי חשוב שהפרויקט לא יעבוד רק "אצלי במחשב", אלא ירגיש כמו מוצר שיכול לצאת לשוק מחר בבוקר.
במקום רק להעלות את הקוד ל-Vercel ולקוות לטוב, בניתי תהליך עבודה אמיתי של פיתוח: הוספתי טסטים (עם Vitest), והגדרתי CI/CD דרך GitHub Actions. ככה, כל פעם שאני מעלה קוד חדש, המערכת קודם כל מריצה את הבדיקות כדי לוודא ששום דבר לא נשבר, ורק אז מעלה את הגרסה החדשה לאוויר. 
בנוסף, לא התפשרתי על אבטחת מידע – הכל מוגן ברמת מסד הנתונים עם Row Level Security (RLS) של Supabase, כך שאף משתמש לא יכול למחוק או לערוך מידע של מישהו אחר.

## 2. מפה חיה בלי להקליד ידנית (שימוש חכם ב-AI)
אחד האתגרים הכי גדולים היה להפוך את המפה למעניינת ורלוונטית מהרגע הראשון. לא רציתי להציג מפה ריקה למשתמשים החדשים או להקליד נתוני דמו חסרי משמעות כמו "test 123".
במקום זה, בניתי סקריפט ייעודי (`seed_direct_ai.js` ו-`seed_wikidata.js`) שמשתמש במודלי שפה (LLMs) ובמשיכת נתונים מויקיפדיה כדי לייצר תוכן אוטומטי, היסטורי ומכובד על אתרי הנצחה ומסלולי מורשת. הסקריפט הזה עיבד את הנתונים, הפעיל אלגוריתמים של קיבוץ מיקומים (Clustering), והזריק הכל ישירות ל-Database. ככה יצרתי "יש מאין" עשרות נקודות ציון ומסלולים שנראים ומרגישים אמיתיים לגמרי.

## 3. חוויית הפיתוח (Vibe Coding)
לאורך כל הפרויקט לא עבדתי לבד, אלא השתמשתי בכלי AI כ"צוות פיתוח" לכל דבר:
* **בקאנד וארכיטקטורה:** השתמשתי ב-Supabase AI Assistant כדי לוודא שסכמות הנתונים שלי מדויקות, לנסח את חוקי האבטחה המורכבים (RLS), ואפילו כדי לתרגם את מבנה הנתונים לתרשים ERD שמצורף להגשה.
* **לוגיקה מורכבת:** בעזרת Claude Code (CLI) שעבד איתי ישירות בתוך הטרמינל, יכולתי להתמודד עם אתגרים מורכבים יותר מהר – למשל, אלגוריתמים שמחשבים מרחקים גיאוגרפיים (נוסחת Haversine) כדי לייצר מסלולים הגיוניים מאנדרטה לאנדרטה.
* **עיצוב ו-UI/UX:** הפינג-פונג המתמיד מול ה-AI איפשר לי ללטש את רכיבי ה-Tailwind, לפתור באגים מעצבנים של תמיכה בעברית (RTL), ולדאוג שחוויית המשתמש במובייל תהיה חלקה וטבעית.

בסופו של דבר, גישת ה-Vibe Coding עזרה לי לא רק לכתוב קוד מהר יותר, אלא להתרכז במה שחשוב באמת – בארכיטקטורה, בחוויית המשתמש, ולהרים פרויקט ברמה שאני באמת גאה בו.2
